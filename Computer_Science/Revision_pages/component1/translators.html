Translators

Three main types - Interpreters, compilers and assemblers

Purpose

CPUs only understand machine code - binary sequences representing instructions and data. The sequences representing instructions are called opcodes. Programming in machine code is not very pleasant due to the fact that humans are better at working with languages, rather than binary number sequences. For this reason, manufacturers provide an assembly language. Assembly language is known as a low-level language. Each command in assembly language is called a ‘mnemonic’. A mnemonic usually has a direct equivalent in machine code, so an assembler is very close to the hardware detail within the CPU such as specific register names and memory management modes.

Assemblers

An assembler is a program that takes basic computer instructions and converts them into a pattern of bits that the computer’s processor can use to perform its basic operations. An assembler is a program that converts assembly language into object code. There is usually a one-to-one relationship between assembly and object code. This means that on the simplest level an assembler just needs to translate each line of code into its binary equivalent. In common with machine code, assembly code in not very portable because it will only run on the CPUs which support those commands.

Compiler

High level languages were developed in order to remove the agony of developing in low level languages. In high level languages eg. C++, C etc. a programmer writes a series of statements in order to create a program. This is known as source code. The purpose of a ‘compiler’ is to translate source code into machine code. A compiler is a program that takes a program written in a high-level language and converts it to object code. The code a compiler generates is stored as an ‘executable file’. Another name for this is ‘object file’. The compiler must also know the target CPU before it is able to translate. Object code can then be distributed and run as many times as required. Advantages of an executable: 1. An executable runs much faster than code which is interpreted line-by-line. 2. The original source code is no longer present and therefore, protects intellectual property. 3. Difficult to make unauthorised changes to an executable.

Compilers also have many features designed to help programmers. Handling errors - will spot and point out errors. Although compilers can detect errors at source code level, often errors are reported which do not exist. These are known as spurious errors. This happens because the compiler has to make assumptions about the rest of the code after detecting an initial error. Because of the assumptions, one real error might cause a whole cascade of other spurious error messages to appear in the report. Code optimisation - There are many different ways of translating a set of high level statements into machine code. Good compilers will try to optimise code by making it run faster or use less memory. Makes source code independent - As the compiler translates, source code written once, can be compiled to work with many different CPUs, thus making it portable and independent of a specific CPU.

Interpreters

An interpreter translates and executes source code line by line. It will translate each line of source code into machine language and then execute that machine code directly eg. a BASIC interpreter command line. A second type of interpreter is one which compiles source code into an intermediate set of instructions known as byte codes. These are then executed by the interpreter. Examples include Flashplayer's action script or Java. Interpreters take each line of a high level program and converts it into machine code. This means that code can be run instantly and can aid in debugging as code will run until it reaches a bug, thus making the job of finding the problem easier. On the other hand, interpreted code is much slower than its compiled alternative. This is because each line must be interpreted before execution. In the case of loops, the same lines may need to be converted to machine code over and over again. Advantages: 1. Debugging is easier. 2. Uses less memory because only a few lines of code in memory at one time. 3. Less prone to crashes because interpreted code is usually run inside a virtual machine eg. JVM. Disadvantages: 1. Slower. Examples of interpreted languages include Python, BASIC and Java.

Compilers: - Create fast, executable code to run on the CPU. - Debugging harder as spurious errors are frequent. - More likely to crash as runs on the CPU. - Easier to protect intellectual property. - Need more memory. - Harder to make unauthorised changes to compiled code.

Interpreters: - Slower as code is interpreted one line at a time. - Debugging easier as code is checked before execution. - Less likely to crash as the instructions are being carried out either on the interpreters' command line or within a virtual machine environment which is protecting the computer from being directly accessed by the code. - Source code easier to obtain. - Uses less memory as code must only be present one line at a time. - Easier to modify.
