Processor performance

Many different types of CPU architectures exist and are used for various applications.

Multi-core and Parallel processing

A CPU can have multiple cores, which are simply CPUs which communicate with each other. A developer can make use of extra cores in order to run some intensive operations using multiple cores at once, thus potentially improving performance. However, writing apps to take advantage of multiple cores is very complex, and may even result in worse performance. If software is not optimised to take advantage of a multi-core CPU, then there is no speed advantage over a single-core CPU.

Clock-speed

A CPU's clock is what syncronises all operations in a CPU. The higher the clock-speed of a CPU, the more operations the CPU will carry out per second. For example, a 3GHz CPU will perform 3 billion cycles per second. However, the higher the clock-speed of a CPU, the more heat is generated. It is therefore inadvisable to overclock a CPU because this may shorten its life, and may result in corrupted data because the normal speed at which data is handled may not be able to keep up with the speed of the CPU.

Cache

The size and amount of Cache available to the CPU will have a great impact on the performance of a CPU. The more fast cache which is available, the higher the chance that a piece of data or instruction will be found in the cache (cache hit).

Pipelining

Processor pipelining can help to improve the performance of a CPU because it can help to reduce to time a CPU sits idle, for example, waiting for the next instruction to arrive. Pipelining means that more things can be done at once, like fetching the next instruction while the current one is being executed.

CPU type

The type of CPU used can affect the performance. For example, a RISC CPU for complex apps may be slower than a CISC CPU.
