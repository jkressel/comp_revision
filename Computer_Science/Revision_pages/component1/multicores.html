Types of multicore CPU

Parallel systems

At some point it is no longer possible to fit more and more transistors on a chip. For this reason, parallel cores offer a solution. By having multiple cores next to each other and talking to each other means that tasks can be broken down into smaller sub-tasks or threads which the various cores can work on at the same time.

Main types of parallel systems - MISD - multiple instruction sets working on the same set of data. SIMD - a single instruction set working on multiple sets of data, eg real time image processing etc. MIMD - Multiple instruction sets working on multiple sets of data. Instructions can be processed independently of other processors. The most common type.

Different processors in a parallel system all work and load instructions as a single core CPU would, the difference being that they can coordinate with each other. Parallel systems are far more complicated to write software for and if not done properly, may not have any performance advantage as was often the case in the early days.

Co-processors

Also work off the principle of adding extra processors but here, instead of having cores which can do everything, cores which are good at specific tasks. Common examples include Floating point units, built in to the CPU to handle more complex FP arithmetic, Digital Signal Processors, commonly used in sound systems and GPUs. GPUs have many uses besides processing the complex calculations needed for graphics. GPUs are used in medical modelling, economic modelling and even password cracking.
