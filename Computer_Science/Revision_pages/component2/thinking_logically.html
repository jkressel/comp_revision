Thinking Logically

Needed to identify points in an algorithm where logical decisions are needed.

What and why?

Logic is the use of logical operators to reason the outcome of a given argument. For example, all possible outcomes of the proposition ‘Natasha will go outside if it is sunny and not raining’ can be calculated using the expression A = B AND NOT(C) where A represents the statement ‘go outside’, B represents ‘is sunny’ and C represents ‘is raining’. The value of A (either TRUE or FALSE) can be determined by applying the logical operators EQUALS, AND, NOT to the values of B and C. Thinking logically is the process of combining logical operators and data sets to produce logical expressions that represent complex arguments and can be used to identify possible outcomes.

Logical operators

> - Greater than.

< - Less than.

=> - Greater than or equal to.

=< - Less than or equal to.

!= - Not equal to.

NOT - Opposite of.

EQUALS (==) - Equal to.

AND - Both statements must be true to return true.

OR - At least one of the statements must be true to return true.

XOR (Exclusive OR) - Only one can be true ie. if both true or both false, then returns false.

How?

Decisions are made during a SELECTION in a program, these usually take the form of IF statements, FOR and WHILE loops. Next, the points in an algorithm where decisions are taken must be identified, one could use a flow diagram to help with this. Once the flow diagram is complete, this can easily be translated into pseudocode (better programming languages are available, my favourite being C). It is also important to determine how logic will affect the flow of a program. It can be useful to write out decisions which must be taken in english, or create a flow diagram so that one can see how decisions affect the flow and result of a program.
